# جدول ما تحتاجون تجهيزه

## 1. متغيرات البيئة (`.env.local` محلياً + Cloudflare/GitHub للإنتاج)

| المتغير | مطلوب؟ | أين يُستخدم | ماذا تضع |
|---------|--------|-------------|----------|
| `NEXT_PUBLIC_SUPABASE_URL` | **نعم** (للتشغيل الكامل) | الدعوات، RSVP، لوحة المالك | رابط مشروع Supabase |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | **نعم** | تسجيل دخول المالك + العميل | Anon key من Supabase |
| `SUPABASE_SERVICE_ROLE_KEY` | **نعم** (سيرفر فقط) | APIs، seed، موافقات | Service role (لا يُعرض للعميل) |
| `NEXT_PUBLIC_SITE_URL` | **موصى به** | روابط QR والتذاكر المطلقة | `https://your-domain.com` |
| `NEXT_PUBLIC_WHATSAPP_URL` | موصى به | أزرار «اطلب دعوتك» والفوتر | `https://wa.me/971XXXXXXXXX` |
| `NEXT_PUBLIC_CONTACT_EMAIL` | اختياري | الفوتر + صفحة الخصوصية | `hello@marasim.ae` |
| `NEXT_PUBLIC_INSTAGRAM_URL` | اختياري | الفوتر | رابط حساب إنستغرام |
| `SEED_OWNER_EMAIL` / `SEED_OWNER_PASSWORD` | تطوير فقط | `npm run db:seed` | حساب ديمو المالك |

**ملاحظة إنتاج (Cloudflare):**  
`NEXT_PUBLIC_*` في GitHub Secrets **و** Worker vars عند النشر.  
`SUPABASE_SERVICE_ROLE_KEY` كـ Worker **secret** فقط.

| ماذا | أين تُعدَّل | ملاحظة |

# صور التصنيفات 

تم الربط.

### أين تضع الصور؟

ارفع الملفات إلى:

`marasim/public/assets/landing/occasions/`

| الملف | المناسبة |
|--------|---------|
| `weddings.webp` | الأعراس |
| `engagement.webp` | الخطوبة |
| `newborns.webp` | المواليد |
| `graduation.webp` | التخرج |
| `private.webp` | مناسبات خاصة |
| `corporate.webp` | فعاليات الشركات |

### مصدر الروابط (مكان واحد)

عدّل المسارات في `src/data/landing-assets.ts` فقط:

```ts
occasionWeddings: "/assets/landing/occasions/weddings.webp",
occasionEngagement: "/assets/landing/occasions/engagement.webp",
// ...
```

النسختان العربية والإنجليزية تستخدمان نفس الصور عبر `imageKey`.

### السلوك

- **إذا وُجدت الصورة:** تظهر فوق البطاقة مع تظليل خفيف ونص أبيض.
- **إذا لم تُرفع أو فشل التحميل:** يبقى **التدرج اللوني الحالي** مع النص الداكن كما كان.

حالياً المجلد فارغ (`.gitkeep` فقط)، لذلك ستشاهد التدرجات إلى أن ترفع الصور.
